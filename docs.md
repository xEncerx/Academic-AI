# Правила использования шаблона ГОСТ

Документ описывает функции и макросы шаблона Typst, а также правила их применения при верстке.

## Основной стиль (`gost-core.typ`)

### Инициализация документа

Функция `gost-core` задает базовые настройки страницы (формат А4, поля), нумерацию, абзацные отступы (1,25 см), полуторный интервал, а также форматирование заголовков и списков.

**Пример вызова в `main.typ`:**

```typst
#import "settings/gost_core.typ": gost-core

#show: gost-core

= НАЗВАНИЕ ГЛАВЫ
Текст главы...

== Подраздел
Текст подраздела...

- Элемент маркированного списка;
- Второй элемент.

+ Элемент нумерованного списка.
+ Второй элемент.
```

### Автоматическое форматирование

При подключении `gost-core` применяются следующие правила:

1. **Параметры страницы:** Формат А4 с полями (левое — 30 мм, правое — 10 мм, верхнее и нижнее — 20 мм). Нумерация сквозная, однако номера (шрифт Times New Roman, 12 pt) выводятся по центру нижнего колонтитула только начиная с шестой страницы.
2. **Заголовки:**
   - **Уровень 1 (`=`):** Начинаются с новой страницы. Шрифт 18 pt, полужирный, все буквы прописные. Номер выводится без точки в конце (например, `1 НАЗВАНИЕ`). Абзацный отступ — 1,25 см.
   - **Уровень 2 (`==`):** Шрифт 16 pt, полужирный. Интервалы до и после текста составляют 15 мм и 10 мм соответственно. Номер оканчивается точкой (`1.1. Название`).
   - **Уровни 3 и ниже (`===`):** Шрифт 14 pt, полужирный.
3. **Основной текст:** Гарнитура Times New Roman, размер 14 pt, выравнивание по ширине. Отступ первой строки установлен на 1,25 см, межстрочный интервал — полуторный.
4. **Списки:** Для маркированных списков (`-`) используется символ `●`, для нумерованных (`+`) — арабские цифры без точки. Отступ маркера или цифры равен 1,25 см, текст начинается на расстоянии 2,25 см от левого края.

> **Важно:** Инструменты Typst управляют отступами, однако пунктуацию и регистр слов автор контролирует самостоятельно. Маркированный список принято начинать со строчной буквы и завершать точкой с запятой. Нумерованный список начинается с прописной буквы и оканчивается точкой.

## Оформление рисунков (`gost_figures.typ`)

Модуль `gost-figures` приводит отображение иллюстраций в соответствие с требованиями ГОСТа. Номера генерируются автоматически, перед названием подставляется слово «Рисунок» и длинное тире. Разрыв рисунка и его подписи между страницами заблокирован.

### Вставка рисунка

Для размещения иллюстраций применяется стандартная функция Typst `#figure()`, поскольку она корректно интегрируется с механизмами нумерации и перекрестных ссылок.

По умолчанию включена сквозная нумерация. Если требуется нумерация по разделам (формата 1.1, где первая цифра обозначает главу, а вторая — порядковый номер рисунка внутри нее), необходимо передать параметр `chapter-numbering: true`.

```typst
// Сквозная нумерация (по умолчанию)
#import "settings/gost_figures.typ": gost-figures
#show: gost-figures

// Нумерация по разделам (Глава.Рисунок)
#import "settings/gost_figures.typ": gost-figures
#show: gost-figures.with(chapter-numbering: true)

// Вставка изображения
#figure(
  image("assets/images/photo.jpg", width: 80%),
  caption: [Детали прибора]
) <img-details>
```
*Результат под изображением: **Рисунок 1 — Детали прибора*** (без точки в конце, шрифт Times New Roman, 12 pt, полужирный, выравнивание по центру).

### Ссылки на рисунки в тексте

Перекрестные ссылки создаются с помощью встроенной функции `#ref()`. Typst распознает тип объекта и автоматически генерирует нужное слово с номером:

```typst
В соответствии с @img-details можно заметить...
// Вывод: В соответствии с Рисунок 1 можно заметить...

// Для изменения падежа передается аргумент supplement:
В соответствии с #ref(<img-details>, supplement: "Рисунком") можно заметить...
// Вывод: В соответствии с Рисунком 1 можно заметить...
```

### Альбомная ориентация 

Для размещения широких графиков страницу необходимо повернуть. По ГОСТу это достигается переходом в альбомную ориентацию:

```typst
#page(flipped: true)[
  #figure(
    image("assets/images/wide_chart.jpg", width: 100%),
    caption: [Широкий график]
  ) <img-wide>
]
```

## Оформление формул (`gost_math.typ`)

Модуль отвечает за верстку математических выражений. Формулы нумеруются по правому краю в круглых скобках, а для расшифровки переменных предусмотрен специальный макрос.

```typst
#import "settings/gost_math.typ": gost-math, gost-where
#show: gost-math.with(chapter-numbering: true)
```

### Формулы и нумерация

По умолчанию нумеруются все вынесенные в отдельный блок формулы. Размер символов вычисляется автоматически на основе базового шрифта 14 pt.

```typst
Формула с номером:
$ Delta T = T_0 - T_1, $ <eq-trud>

По правилам нумеровать следует только те формулы, на которые есть ссылки в тексте.
Для отключения номера используйте локальную настройку:
#block[
  #set math.equation(numbering: none)
  $ A = B + C $
]
```

### Перекрестные ссылки на формулы

Для вызова номера в тексте применяется `#ref()`. Макрос самостоятельно обернет номер в круглые скобки:

```typst
Расчет производится по формуле @eq-trud.
// Вывод: Расчет производится по формуле (7.1).
```

### Пояснения к переменным

Список значений символов оформляется макросом `gost-where`. Каждая переменная передается как массив из двух элементов `(символ, [описание])`. Макрос добавляет слово «где», проставляет тире и выравнивает текст без абзацного отступа.

```typst
$ Delta T = T_0 - T_1, $ <eq-trud>
#gost-where(
  ($T_0$, [трудовые затраты на обработку информации по базовому варианту;]),
  ($T_1$, [трудовые затраты на обработку информации по предлагаемому варианту.])
)
```

> **Важно:** Следите за знаками препинания. Запятая ставится после формулы, точка с запятой — между элементами списка пояснений, точка — в конце. При переносе длинного математического выражения на новую строку знаки равенства или арифметических операций дублируются вручную: `$ a = b + c + \ + d $`

## Оформление таблиц (`gost_tables.typ`)

Модуль `gost-tables` автоматизирует форматирование табличных данных. Он обеспечивает выравнивание подписи над таблицей (`Таблица 1.1 — Название`, шрифт 12 pt, по левому краю) и устанавливает одинарный интервал для содержимого без абзацных отступов. Заголовки столбцов центрируются и выделяются полужирным начертанием, в то время как обычные ячейки выравниваются по левому краю.

При разрыве таблицы на следующую страницу макрос переносит заголовки и автоматически добавляет надпись вида *«Продолжение Таблицы N»* в правом верхнем углу.

```typst
#import "settings/gost_tables.typ": gost-tables, gost-table
#show: gost-tables.with(chapter-numbering: true)
```

### Использование макроса `#gost-table`

Для таблиц, занимающих более одной страницы, применяется функция `#gost-table`. 

**Основные параметры:**
- `columns`: Ширина столбцов в виде массива (допускаются значения `auto` или `1fr`).
- `caption`: Название таблицы.
- `header`: Массив заголовков столбцов.
- `..cells`: Данные ячеек.

```typst
#gost-table(
  columns: (1fr, 2fr, 2fr),
  caption: [Анализ параметров],
  header: ([Параметр], [Значение 1], [Значение 2]),
  [Шрифт], [Times New Roman], [Arial],
  [Интервал], [Обычный], [Полуторный]
) <tab-analysis>
```

> **Особенности:** На данный момент механизм Typst сохраняет нижнюю рамку таблицы при ее разрыве в конце страницы. Текст *«Продолжение Таблицы N»* формируется автоматически. Выравнивание заголовков строк и данных в графах при необходимости корректируется вручную.

## Листинги программного кода (`gost_listings.typ`)

Исходный код оформляется с обводкой тонкой черной рамкой и моноширинным шрифтом (Courier New, 10 pt). Подпись располагается над блоком (12 pt, курсив, выравнивание по левому краю). Для ссылок на фрагменты кода требуется нумерация строк с шагом 10.

```typst
#import "settings/gost_listings.typ": gost-listings, gost-listing
#show: gost-listings.with(chapter-numbering: true)
```

### Вставка листинга

Функция `#gost-listing()` считывает код из внешнего файла или напрямую из строки, расставляя нумерацию.

**Параметры:**
- `file`: Путь к исходному файлу (например, `"assets/codes/script.py"`).
- `code`: Строка с кодом (если файл не используется).
- `caption`: Название блока.
- `lang`: Язык для подсветки синтаксиса (опционально).
- `numbers`: При значении `true` выводит номера строк слева (`10, 20, 30...`).
- `new-page`: При `true` переносит листинг на новую страницу (часто используется для приложений).

**Пример вставки из файла:**
```typst
#gost-listing(
  file: "assets/codes/main.py",
  caption: [Скрипт запуска],
  lang: "python",
  numbers: true
) <list-main>
```

### Ссылки на строки

Для указания конкретных строк в тексте номера перечисляются в фигурных скобках после метки:

```typst
В @list-main {30, 50, 140} показаны основные вызовы.
Как видно из @list-main {20-80}, алгоритм...
```
*(Результат: «В Листинг 1.7 {30, 50, 140} показаны...»)*

## Титульный лист и Содержание (`gost_intro_frame.typ`)

Модуль предназначен для встраивания готового PDF-файла обложки и автоматической генерации оглавления.

```typst
#import "settings/gost_intro_frame.typ": gost-title, gost-toc
```

### Вставка титульного листа

Готовый титульный лист можно подложить первой страницей документа без полей и колонтитулов. Макрос `#gost-title()` вызывается в начале `main.typ`, строго до подключения основного стиля `gost-core`.

```typst
// Вставка PDF-файла (по умолчанию берется из "assets/title_page.pdf")
#gost-title()

// Указание альтернативного пути:
#gost-title(path: "assets/my_title.pdf")

#show: gost-core // Подключение стилей документа
```

### Генерация содержания

Оглавление собирается из заголовков трех верхних уровней. Слово «СОДЕРЖАНИЕ» печатается шрифтом 18 pt, полужирным начертанием, без нумерации и выравнивается по центру. Заголовки первого уровня автоматически переводятся в верхний регистр.

```typst
#gost-toc()
```

> **Перенос строк:** Если название пункта перекрывает номера страниц в содержании, разрыв строки `\ ` проставляется вручную в самом заголовке по тексту документа. Typst автоматически перенесет это форматирование в оглавление.

## Список использованных источников (`gost_references.typ`)

Оформление библиографии с разбивкой по категориям (Аналитический, Специальный разделы) и составной нумерацией реализуется через набор макросов.

```typst
#import "settings/gost_references.typ": gost-bibliography-title, gost-bib-section
```

### Разделы списка

1. Вызов `#gost-bibliography-title()` формирует новую страницу с отцентрированным заголовком «СПИСОК ИСПОЛЬЗОВАННЫХ ИСТОЧНИКОВ».
2. Функция `#gost-bib-section(номер, "Название")` создает подзаголовок раздела с нужными отступами.

```typst
#gost-bibliography-title()

#gost-bib-section(1, "Аналитический раздел")

1.1. Борзенко А. и др. Компьютерная азбука // Компьютер-пресс. — 2006. — № 11. — С. 124–130.

#gost-bib-section(2, "Специальный раздел")

2.1. Надарая Э. А. Об оценке регрессии // Теория вероятностей и ее применения. — 2000. — Т. 9. — Вып. 1. — С. 157–159.
```

Сами элементы списка верстаются обычными абзацами — базовые настройки стиля обеспечат правильный отступ первой строки и выравнивание. Внутритекстовые ссылки оформляются вручную в квадратных скобках: `[1.2, с.21-25]`.

> **Пунктуация:** Не ставьте пробелы после знаков препинания в сокращениях городов (например: `М.:` или `СПб.:`), а также между инициалами (`Иванов И.И.`). Для разделения областей библиографической записи используется длинное тире `—`.

## Приложения (`gost_appendix.typ`)

Приложения группируются в конце документа. Они перечисляются на отдельной странице, нумеруются буквами (Приложение А, Приложение Б) и имеют собственные счетчики для внутренних объектов (Рисунок А.1).

```typst
#import "settings/gost_appendix.typ": gost-appendix-list, gost-appendix
```

### Перечень и структура

Страница-перечень создается макросом `#gost-appendix-list`. В него передаются кортежи с буквой и названием.

```typst
#gost-appendix-list(
  ("А", "Техническое задание"),
  ("Б", "Листинги исходного кода")
)
```

Каждое приложение оборачивается в функцию `#gost-appendix`. При ее вызове счетчики таблиц, рисунков и листингов сбрасываются и адаптируются под буквенную нумерацию.

```typst
#gost-appendix("А", "Техническое задание")[
  Текст технического задания.

  #figure(image("scheme.png"), caption: [Схема]) <img-a>
  // Результат: "Рисунок А.1 — Схема"
]
```
